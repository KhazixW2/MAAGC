# 连接设置

本文档详细介绍如何配置 MaaGC 与游戏设备的连接。

## 📱 一、模拟器连接（推荐）

### 1.1 支持的模拟器

MaaGC 支持所有开启 ADB 调试的安卓模拟器：

- ✅ MuMu 模拟器 (推荐)
- ✅ 雷电模拟器 9
- ✅ 夜神模拟器
- ✅ 蓝叠模拟器
- ✅ 逍遥模拟器
- ✅ 其他支持 ADB 的模拟器

### 1.2 模拟器配置步骤

#### 步骤 1：开启 ADB 调试

**雷电模拟器 9**：

1. 打开雷电模拟器
2. 点击右上角"设置"（齿轮图标）
3. 进入"其他"选项卡
4. 勾选"开启 ROOT 权限"和"开启 ADB 调试"
5. 保存设置并重启模拟器

**夜神模拟器**：

1. 打开夜神模拟器
2. 点击右上角"设置"
3. 进入"常规"选项卡
4. 勾选"开启 ROOT 权限"和"ADB 调试"
5. 保存设置

**MuMu 模拟器**：

1. 打开 MuMu 模拟器
2. 点击右上角"菜单"
3. 选择"设置中心"
4. 进入"其他"选项卡
5. 开启"ADB 调试"

#### 步骤 2：获取 ADB 端口

不同模拟器默认 ADB 端口：

| 模拟器 | 默认端口 | 备注 |
|-------|---------|------|
| 雷电 9 | 5555 | 最常用 |
| 夜神 | 62001 | 多开时递增 |
| MuMu | 7555 | 部分版本 5555 |
| 蓝叠 | 5555 | - |

**查看端口方法**：

1. 打开模拟器设置
2. 查找"ADB 端口"或类似选项
3. 记录端口号

#### 步骤 3：验证 ADB 连接

打开命令行（CMD 或 PowerShell），输入：

```bash
adb connect 127.0.0.1:5555
adb devices
```

如果看到设备列表中有设备，说明连接成功。

### 1.3 在 MaaGC 中配置

#### MFAA/MXU 版本

1. 打开 MaaGC
2. 点击"连接设置"或"配置"
3. 选择控制器类型为"ADB"
4. 输入 ADB 地址：`127.0.0.1:5555`
5. 点击"连接"或"测试连接"
6. 连接成功后会显示设备信息

#### PiCLI 版本

创建或编辑配置文件 `config.json`：

```json
{
  "controller": {
    "type": "Adb",
    "address": "127.0.0.1:5555"
  }
}
```

### 1.4 多开配置

如需同时运行多个模拟器实例：

**配置示例**：

| 实例 | ADB 端口 | MaaGC 配置 |
|-----|---------|-----------|
| 实例 1 | 5555 | 127.0.0.1:5555 |
| 实例 2 | 5556 | 127.0.0.1:5556 |
| 实例 3 | 5557 | 127.0.0.1:5557 |

**注意事项**：

- 每个实例需要独立的 MaaGC 配置
- 使用不同的配置文件夹
- 分别启动和管理

## 💻 二、PC 端连接

### 2.1 系统要求

- Windows 10/11
- 游戏已安装并可正常运行
- 游戏窗口可正常显示

### 2.2 配置步骤

#### 步骤 1：启动游戏

1. 正常启动游戏客户端
2. 等待游戏进入主界面
3. 确保游戏窗口可见

#### 步骤 2：选择窗口

**MFAA/MXU 版本**：

1. 打开 MaaGC
2. 选择控制器类型为"Win32"
3. 从下拉列表中选择游戏窗口
4. 点击"连接"

**PiCLI 版本**：

```json
{
  "controller": {
    "type": "Win32",
    "window_name": "游戏窗口标题"
  }
}
```

### 2.3 窗口要求

- 游戏窗口必须在前台运行
- 窗口不能被最小化
- 避免其他窗口遮挡游戏窗口
- 建议使用窗口模式而非全屏

### 2.4 分辨率设置

**推荐分辨率**：

- 720x1280（推荐）
- 1920x1080
- 其他 9:16 比例分辨率

**调整方法**：

1. 打开游戏设置
2. 进入"画面设置"
3. 选择合适的分辨率
4. 保存设置

## 🔍 三、连接问题排查

### 问题 1：无法连接到模拟器

**可能原因**：

- ADB 调试未开启
- 端口号错误
- 模拟器未完全启动
- 防火墙阻止连接

**解决方案**：

1. 确认模拟器已开启 ADB 调试
2. 检查端口号是否正确
3. 等待模拟器完全启动
4. 暂时关闭防火墙测试
5. 重启模拟器和 MaaGC

### 问题 2：连接后无法识别画面

**可能原因**：

- 分辨率不符合要求
- 游戏语言不是简体中文
- 画面被遮挡
- OCR 资源未正确加载

**解决方案**：

1. 调整分辨率为 720x1280
2. 设置游戏语言为简体中文
3. 确保游戏窗口完整显示
4. 重新下载 OCR 资源文件

### 问题 3：连接不稳定

**可能原因**：

- 网络问题
- 模拟器性能不足
- ADB 版本过旧

**解决方案**：

1. 使用本地模拟器（127.0.0.1）
2. 增加模拟器内存分配
3. 更新 ADB 到最新版本
4. 关闭不必要的后台程序

## 🛠️ 四、高级配置

### 4.1 自定义 ADB 路径

如果模拟器自带的 ADB 不可用，可以使用系统 ADB：

**MFAA/MXU 版本**：
在设置中指定 ADB 可执行文件路径

**PiCLI 版本**：

```json
{
  "controller": {
    "type": "Adb",
    "adb_path": "C:\\path\\to\\adb.exe"
  }
}
```

### 4.2 无线连接

部分模拟器支持无线 ADB 连接：

```bash
# 启用无线调试
adb tcpip 5555

# 获取设备 IP
adb shell ip addr show wlan0

# 无线连接
adb connect 设备 IP:5555
```

### 4.3 多设备管理

使用 ADB 管理多个设备：

```bash
# 查看所有连接的设备
adb devices

# 指定设备执行命令
adb -s 设备序列号 shell

# 断开设备
adb disconnect 设备 IP:端口
```

## 📊 五、性能优化

### 5.1 模拟器优化

**CPU 设置**：

- 分配 2-4 核心
- 开启虚拟化技术（VT）

**内存设置**：

- 分配 2-4GB 内存
- 根据系统内存调整

**渲染设置**：

- 使用 DirectX 或 OpenGL 模式
- 关闭不必要的特效
- 降低帧率到 30FPS

### 5.2 系统优化

- 关闭不必要的后台程序
- 设置高性能电源模式
- 更新显卡驱动
- 确保足够的磁盘空间
